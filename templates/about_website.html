Library Management System

Project Overview

This project implements a simple system to manage a library of books. It was developed based on the following requirements:

  • Create a SQLite database with three tables: Books, Customers, and Loans.
  • Define the book type to set the maximum loan time for each book using an Enum: 1 (up to 10 days), 2 (up to 5 days), and 3 (up to 2 days).
  • Utilize SQLAlchemy to build classes for each entity, organize the code into separate modules, and create unit tests.
  • Develop a client application that allows users to manage their book loans.
  • Upload the project to GitHub and deploy it using Render.
How the Website Works

The library is designed for readers who can browse a list of available books, add them to their cart, and view the details of their selected loans. Additionally, authors can publish their books and offer them for reading. Site administrators have access to information about all customers and can monitor books currently on loan, ensuring timely returns.
